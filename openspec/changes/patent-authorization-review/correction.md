# 提案修正说明

**变更ID**: patent-authorization-review
**修正日期**: 2026-01-19

## 重要声明

本提案旨在建立一个**内部审查流程**,用于在提交专利申请前识别潜在问题。以下几点需要明确:

### 1. 关于数据来源

**原提案中的问题**:
- 引用了"发明专利平均授权率60-70%"等数据,但未提供来源
- 这些数据可能不准确或已过时

**修正**:
- **删除所有未经验证的统计数据**
- 仅使用可验证的法律条文和官方文件
- 对于行业经验,明确标注为"经验估计"而非"事实数据"

### 2. 关于审查标准

**明确区分**:

#### 官方标准(基于法律法规)
- 《中华人民共和国专利法》(2020年修正)
- 《专利审查指南》(2010年版,2023年修订)
- 这些是**客观的法律要求**,必须严格遵守

#### 内部标准(我们自己制定)
- 五维评分框架 - **这是我们的内部工具**,不是官方标准
- 100分制评分 - **这是我们的内部评估方法**,不代表专利局的评判
- 授权成功率预估 - **这是基于经验的主观判断**,不是精确预测

### 3. 关于审查工具

**实际情况**:
- 专利审查是**高度专业化的人工工作**
- 自动化工具只能**辅助检查**,不能替代人工判断
- 工具的作用:
  - ✅ 检查格式规范
  - ✅ 检查必要章节是否完整
  - ✅ 检索现有技术文献
  - ❌ 不能判断创造性
  - ❌ 不能判断技术效果是否显著
  - ❌ 不能替代专业代理人的审查

### 4. 关于改进建议

**改进建议的依据**:
1. **法律条文**: 基于《专利法》和《专利审查指南》的明确规定
2. **审查案例**: 基于公开的专利审查决定书和复审决定
3. **专业经验**: 基于专利代理实践的经验总结(需明确标注)

**不应包含**:
- ❌ 凭空想象的"可能"问题
- ❌ 未经验证的"最佳实践"
- ❌ 没有法律依据的"建议"

## 修正后的审查方法

### 第一步:法律合规性检查(客观标准)

基于《专利法》第22条和《专利审查指南》,检查:

1. **新颖性检查**:
   - 方法:检索中国专利数据库、外文专利数据库
   - 标准:是否存在相同的技术方案
   - 输出:相同技术方案列表(如有)

2. **创造性评估**:
   - 方法:三步法(《专利审查指南》第二部分第四章)
   - 标准:区别特征是否显而易见
   - 输出:最接近现有技术、区别特征、技术问题

3. **实用性验证**:
   - 方法:检查技术方案是否完整
   - 标准:本领域技术人员能否实施
   - 输出:缺失的技术特征(如有)

4. **客体适格性判断**:
   - 方法:对照《专利法》第25条
   - 标准:是否属于不授予专利权的客体
   - 输出:不符合项(如有)

### 第二步:文档质量检查(客观标准)

基于《专利审查指南》第二部分第二章,检查:

1. **说明书检查**:
   - ✅ 是否包含技术领域、背景技术、发明内容、附图说明、具体实施方式
   - ✅ 是否充分公开(本领域技术人员能否实施)
   - ✅ 实施例是否具体

2. **权利要求书检查**:
   - ✅ 是否清楚、简要
   - ✅ 是否得到说明书支持
   - ✅ 是否包含全部必要技术特征

3. **附图检查**:
   - ✅ 是否符合格式要求(《专利审查指南》第一部分第一章)
   - ✅ 图片是否清晰
   - ✅ 标注是否完整

### 第三步:现有技术检索(客观方法)

使用公开数据库进行检索:

1. **专利数据库**:
   - 中国专利公布公告网(官方)
   - 国家知识产权局专利检索系统(官方)
   - Google Patents(参考)

2. **学术文献**:
   - 中国知网(CNKI)
   - 万方数据
   - IEEE Xplore

3. **开源项目**:
   - GitHub
   - GitLab

4. **检索方法**:
   - 关键词检索
   - IPC分类号检索
   - 申请人检索

### 第四步:问题识别与分类(客观记录)

对发现的问题进行分类:

1. **法律问题**(必须解决):
   - 不符合新颖性
   - 不符合创造性
   - 不符合实用性
   - 属于不授权客体

2. **文档问题**(必须解决):
   - 说明书不充分公开
   - 权利要求不清楚
   - 权利要求得不到支持
   - 附图不符合规范

3. **优化建议**(可选):
   - 权利要求可以更清晰
   - 实施例可以更具体
   - 技术效果可以更量化

### 第五步:改进方案(基于法律要求)

针对识别的问题,提出改进方案:

1. **法律问题的改进**:
   - 依据:《专利法》和《专利审查指南》的明确规定
   - 方法:修改技术方案、增加区别特征、补充技术效果

2. **文档问题的改进**:
   - 依据:《专利审查指南》的撰写要求
   - 方法:补充说明、修改权利要求、完善附图

3. **优化建议的实施**:
   - 依据:专利代理实践经验(明确标注)
   - 方法:细化描述、增加实施例、量化效果

## 修正后的输出

每个专利的审查报告应包含:

### 1. 法律合规性检查结果(客观)
- 新颖性检查:✅通过 / ❌发现相同技术方案
- 创造性评估:✅通过 / ❌区别特征显而易见
- 实用性验证:✅通过 / ❌技术方案不完整
- 客体适格性:✅通过 / ❌属于不授权客体

### 2. 文档质量检查结果(客观)
- 说明书:✅完整 / ❌缺少XX章节
- 权利要求:✅清楚 / ❌存在XX问题
- 附图:✅符合规范 / ❌存在XX问题

### 3. 现有技术检索结果(客观)
- 相关专利:列表(专利号、标题、相关度)
- 相关文献:列表(标题、作者、发表时间)
- 相关开源项目:列表(项目名、链接、相关度)

### 4. 问题清单(客观)
- 法律问题:列表(问题描述、法律依据)
- 文档问题:列表(问题描述、规范依据)

### 5. 改进建议(明确标注依据)
- 必须改进:列表(改进内容、法律依据)
- 建议优化:列表(优化内容、经验依据)

### 6. 风险评估(主观判断,明确标注)
- **声明**:以下评估基于审查经验,不代表专利局的最终判断
- 高风险问题:列表
- 中风险问题:列表
- 低风险问题:列表

## 关键原则

1. **事实与判断分离**:
   - 事实:法律条文、检索结果、文档内容
   - 判断:风险评估、改进建议、成功率预估

2. **来源可追溯**:
   - 所有法律依据注明条款
   - 所有检索结果提供链接
   - 所有经验判断明确标注

3. **保守估计**:
   - 不夸大授权成功率
   - 不低估改进难度
   - 不承诺100%授权

4. **专业谦逊**:
   - 承认自动化工具的局限性
   - 建议关键专利咨询专业代理人
   - 最终决策权在用户
