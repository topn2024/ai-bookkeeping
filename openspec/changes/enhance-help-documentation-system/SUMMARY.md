# 提案总结：完善帮助文档系统

## 概述

为AI Bookkeeping应用构建一个系统化的帮助文档系统，覆盖全部20个功能模块和237个页面，提供结构化、易维护、可搜索的帮助内容，帮助用户快速了解和掌握应用功能。

## 核心目标

1. **全覆盖**：为237个应用页面提供完整的帮助文档
2. **易查找**：支持按模块浏览和关键词搜索
3. **易维护**：数据驱动架构，JSON格式存储，便于更新
4. **离线可用**：所有内容随应用打包，无需网络

## 技术方案

### 数据模型
```dart
HelpContent {
  pageId, title, module,           // 基本信息
  description,                      // 功能描述
  useCases, steps,                  // 使用场景和操作步骤
  tips, relatedPages                // 进阶内容
}
```

### 文件组织
- 20个JSON文件，按模块组织（`assets/help/*.json`）
- 每个文件包含5-30个页面的帮助内容
- 版本控制友好，支持多人协作编写

### UI架构
```
HelpPage
├── 搜索栏（全局搜索）
└── TabBar
    ├── 快速入门（精选教程）
    ├── 按模块浏览（20个模块 → 页面列表 → 详情）
    └── 常见问题（FAQ）
```

## 关键特性

### 1. 模块化组织
- 20个功能模块清晰分类
- 每个模块有概述和页面列表
- 支持从模块到页面的层级浏览

### 2. 智能搜索
- 全文搜索：标题、描述、场景、步骤
- 相关度排序：标题匹配优先
- 实时结果：输入即显示

### 3. 快速入门
- 精选5-10个核心教程
- 新用户友好
- 引导式学习路径

### 4. 上下文帮助
- 所有页面右上角显示帮助图标
- 点击直达当前页面帮助
- 无需手动搜索

### 5. 相关功能链接
- 帮助详情页展示相关功能
- 横向导航，发现功能关联
- 提升功能使用深度

## 技术风险及缓解

### 风险1：内容维护成本高
- 缓解：建立开发流程规范，功能PR必须包含帮助更新
- 缓解：CI自动检查帮助内容完整性

### 风险2：JSON文件冲突
- 缓解：按模块拆分20个文件，减少冲突
- 缓解：协调内容编写分工

### 风险3：搜索性能
- 缓解：内容全部缓存在内存（~237KB）
- 缓解：监控性能，必要时优化算法

## 扩展性考虑

### 支持的未来扩展
1. **多语言**：每种语言独立JSON文件集
2. **在线更新**：从服务器下载最新帮助内容
3. **媒体内容**：图片、视频、动图
4. **用户笔记**：允许用户添加个人注释
5. **智能推荐**：根据使用习惯推荐帮助

### 设计原则
- 保持核心模型简单
- 通过可选字段支持扩展
- 避免过度设计
