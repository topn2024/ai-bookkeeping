# 专利文档规范增量

## 新增需求

### 需求：差分隐私参数科学化

专利文档中的差分隐私参数（ε值）必须具备科学依据，包括：
- 理论来源引用
- 敏感度分级说明
- 隐私-效用权衡分析

#### 场景：高敏感度数据ε值设定
- **当** 处理账户余额、银行卡号等高敏感度数据时
- **那么** ε值设定为0.1，并引用Apple Privacy-Preserving Analytics的相关研究

#### 场景：中敏感度数据ε值设定
- **当** 处理消费金额、交易时间等中敏感度数据时
- **那么** ε值设定为0.5，并说明该值在隐私-效用权衡曲线上的位置

#### 场景：低敏感度数据ε值设定
- **当** 处理分类统计、趋势数据等低敏感度数据时
- **那么** ε值设定为1.0，并说明该值符合工业界常见实践

### 需求：专利与现有技术对比表

每项专利必须包含与现有技术的对比表，采用三栏结构：
- 现有技术特征
- 我方技术特征
- 本质区别说明

#### 场景：位置增强专利与EP4506878A2对比
- **当** 撰写P06位置增强管理专利时
- **那么** 必须包含与EP4506878A2的技术对比表，明确说明"预算预警"与"费用报销"的本质区别

#### 场景：可视化专利与通用UI专利对比
- **当** 撰写P08智能可视化专利时
- **那么** 必须包含与通用渐进式披露UI专利的对比表，强调财务领域的特殊优化

### 需求：CRDT财务应用创新说明

离线增量同步专利必须清晰阐述CRDT在财务场景的创新应用：
- 传统CRDT的局限性分析
- 财务语义冲突解决机制
- 精度保护技术说明

#### 场景：财务精度保护说明
- **当** 描述CRDT合并机制时
- **那么** 必须说明Decimal(18,2)精度保护的具体实现，以及如何避免浮点数精度丢失

#### 场景：借贷平衡约束说明
- **当** 描述冲突解决策略时
- **那么** 必须说明如何在分布式环境下维护借贷平衡约束

### 需求：技术效果量化描述

每项专利必须包含可量化的技术效果描述，避免使用"显著改善"等模糊表述。

#### 场景：计算效率量化
- **当** 描述算法效率时
- **那么** 使用具体的复杂度表示（如O(n)→O(1)）或百分比提升

#### 场景：准确率量化
- **当** 描述识别或预测准确率时
- **那么** 使用具体的百分比数值（如"准确率达到95%"）

### 需求：专利组合协同效应说明

专利文档必须包含专利组合的协同效应说明：
- 技术依赖关系图
- 数据流转说明
- 组合使用的增值效果

#### 场景：隐私-健康-同步协同
- **当** 说明P03、P09、P11的关系时
- **那么** 必须描述差分隐私数据如何流向健康评分计算，再通过同步机制分发

#### 场景：导入-去重协同
- **当** 说明P07、P10的关系时
- **那么** 必须描述账单导入与交易去重的双向协作机制
