# 规范：轻量查询卡片

## 新增需求

### 需求：轻量卡片UI组件

系统必须提供轻量级的查询结果卡片组件，用于Level 2响应。

#### 场景：进度卡片显示

**前置条件**：
- 查询结果包含进度数据（如预算使用情况）
- 复杂度判定为Level 2

**操作**：
- 系统构建进度卡片数据
- 显示LightweightQueryCard组件

**预期结果**：
- 卡片显示主要数值
- 显示进度条（0-100%）
- 卡片淡入动画（500ms）
- 3秒后自动淡出

**示例**：
- 查询："餐饮预算还剩多少"
- 卡片显示："已用2180元 / 预算2500元"
- 进度条：87%

#### 场景：占比卡片显示

**前置条件**：
- 查询结果包含占比数据
- 复杂度判定为Level 2

**操作**：
- 系统构建占比卡片数据
- 显示LightweightQueryCard组件

**预期结果**：
- 卡片显示主要数值
- 显示占比百分比
- 卡片淡入动画（500ms）
- 3秒后自动淡出

**示例**：
- 查询："餐饮这个月花了多少"
- 卡片显示："2180元"
- 占比："占总支出的29.3%"

#### 场景：对比卡片显示

**前置条件**：
- 查询结果包含对比数据（环比、同比）
- 复杂度判定为Level 2

**操作**：
- 系统构建对比卡片数据
- 显示LightweightQueryCard组件

**预期结果**：
- 卡片显示本期和上期数值
- 显示变化百分比和方向（↑/↓）
- 卡片淡入动画（500ms）
- 3秒后自动淡出

**示例**：
- 查询："本月和上月对比"
- 卡片显示："本月8400元 / 上月9800元"
- 变化："↓ 14.3%"

### 需求：卡片自动淡出

卡片必须在显示3秒后自动淡出，不阻碍用户操作。

#### 场景：卡片生命周期

**前置条件**：
- 卡片已显示

**操作**：
- 等待3秒

**预期结果**：
- 卡片开始淡出动画（500ms）
- 淡出完成后从界面移除
- 不影响其他UI元素

### 需求：卡片样式规范

卡片必须遵循统一的设计规范。

#### 场景：卡片视觉设计

**前置条件**：
- 显示任意类型的卡片

**操作**：
- 渲染卡片组件

**预期结果**：
- 圆角：8px
- 内边距：16px
- 阴影：轻微阴影
- 背景：半透明白色
- 字体：主要数值使用大号字体，次要信息使用小号字体
