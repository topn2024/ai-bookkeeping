# 手势交互规范增量

## 新增需求

### 需求：对话框滑动关闭

系统必须支持用户通过滑动手势关闭对话框，提供流畅的动画反馈和触觉反馈。

#### 场景：水平滑动关闭对话框

- **当** 用户在对话框上水平滑动超过 100dp
- **那么** 对话框以动画形式关闭，返回 null 值
- **并且** 触发 mediumImpact 触觉反馈

#### 场景：滑动距离不足时回弹

- **当** 用户滑动距离小于 100dp 后松手
- **那么** 对话框以动画形式回弹到原始位置
- **并且** 对话框保持打开状态

#### 场景：滑动过程中的视觉反馈

- **当** 用户正在滑动对话框
- **那么** 对话框随手指移动产生位移
- **并且** 对话框透明度随滑动距离线性降低（最低 50%）

#### 场景：配置滑动方向

- **当** 开发者设置 `direction: SwipeableDismissDirection.horizontal`
- **那么** 仅支持左右滑动关闭
- **当** 开发者设置 `direction: SwipeableDismissDirection.vertical`
- **那么** 仅支持上下滑动关闭
- **当** 开发者设置 `direction: SwipeableDismissDirection.both`
- **那么** 支持任意方向滑动关闭

#### 场景：禁用危险操作的滑动关闭

- **当** 对话框标记为危险操作（isDangerous = true）
- **那么** 默认禁用滑动关闭功能
- **并且** 用户必须明确点击按钮才能关闭
