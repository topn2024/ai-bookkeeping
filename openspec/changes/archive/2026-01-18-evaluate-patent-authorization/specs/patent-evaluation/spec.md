# patent-evaluation 规范

## Purpose

提供专利三性（新颖性、创造性、实用性）评估能力，预测授权通过率并识别改进空间。

## 新增需求

### 需求：数据真实性验证

系统必须提供数据真实性验证功能，确保专利申请材料中的所有数据声明都有可验证的来源。

#### 场景：代码溯源验证

- **当** 验证专利的技术方案
- **那么** 必须建立技术方案与代码的映射关系
- **并且** 验证说明书中的技术描述与代码实现一致
- **并且** 验证算法流程图与代码逻辑一致
- **并且** 记录对应的代码文件路径和函数名

#### 场景：性能数据核查

- **当** 专利中声明性能数据（响应时间、吞吐量等）
- **那么** 必须提供性能测试报告或基准测试代码
- **并且** 无法验证的性能数据必须删除或标注"待验证"

#### 场景：准确率数据核查

- **当** 专利中声明准确率数据（识别率、匹配率等）
- **那么** 必须提供测试集、评估代码和测试结果
- **并且** 无法验证的准确率数据必须删除或标注"待验证"

#### 场景：对比数据核查

- **当** 专利中声明与现有技术的对比数据
- **那么** 必须提供对比测试方法和基准数据
- **并且** 无法验证的对比数据必须改为定性描述

### 需求：现有技术检索

系统必须提供现有技术检索功能，确保申请的专利不与已有专利或公开技术重复。

#### 场景：专利数据库检索

- **当** 评估专利的新颖性前
- **那么** 必须在以下数据库进行检索：
  - 国家知识产权局专利检索系统
  - Google Patents
- **并且** 记录检索关键词和检索结果
- **并且** 分析与现有专利的相似度

#### 场景：相似专利分析

- **当** 发现相似度超过50%的现有专利
- **那么** 必须详细分析技术差异点
- **并且** 提出权利要求调整建议
- **并且** 相似度超过80%时建议放弃申请或重大修改

#### 场景：学术文献检索

- **当** 需要全面评估新颖性
- **那么** 应检索相关学术论文和技术文档
- **并且** 检索开源代码库中的相似实现

### 需求：专利三性评估

系统必须提供专利三性评估功能，对每项专利进行新颖性、创造性、实用性的系统性评估。

#### 场景：新颖性评估

- **当** 评估专利的新颖性
- **那么** 必须分析以下方面：
  - 技术方案与现有技术的差异
  - 核心技术特征的独特性
  - 现有技术检索结果
- **并且** 给出新颖性评分（0-100分）
- **并且** 识别潜在的新颖性风险

#### 场景：创造性评估

- **当** 评估专利的创造性
- **那么** 必须分析以下方面：
  - 技术方案的非显而易见性
  - 与现有技术的技术启示关系
  - 技术效果的显著性
- **并且** 给出创造性评分（0-100分）
- **并且** 识别可能被认为显而易见的技术特征

#### 场景：实用性评估

- **当** 评估专利的实用性
- **那么** 必须分析以下方面：
  - 技术方案的完整性
  - 本领域技术人员的可实施性
  - 技术效果的可验证性
- **并且** 给出实用性评分（0-100分）
- **并且** 识别需要补充的实施细节

### 需求：授权通过率预测

系统必须提供授权通过率预测功能，基于三性评估结果预测专利授权概率。

#### 场景：计算授权概率

- **当** 完成专利三性评估
- **那么** 根据加权公式计算综合得分：
  - 新颖性权重：40%
  - 创造性权重：35%
  - 实用性权重：25%
- **并且** 根据综合得分预测授权等级：
  - A级（85-100分）：高通过率（>85%）
  - B级（70-84分）：中等通过率（60-85%）
  - C级（55-69分）：需改进（40-60%）
  - D级（<55分）：风险较高（<40%）

#### 场景：生成评估报告

- **当** 完成所有专利的评估
- **那么** 生成综合评估报告，包含：
  - 各专利的三性评分
  - 授权预测等级
  - 专利组合整体分析
  - 申请优先级建议

### 需求：专利改进建议

系统必须提供专利改进建议功能，针对评估中发现的问题提出具体改进措施。

#### 场景：权利要求优化建议

- **当** 发现权利要求存在问题
- **那么** 提供优化建议：
  - 保护范围调整建议
  - 从属权利要求细化建议
  - 限制性词语修改建议

#### 场景：说明书完善建议

- **当** 发现说明书存在不足
- **那么** 提供完善建议：
  - 技术细节补充建议
  - 技术效果量化建议
  - 实施例丰富建议

#### 场景：附图改进建议

- **当** 发现附图存在问题
- **那么** 提供改进建议：
  - 系统架构图优化
  - 流程图完善
  - 时序图细化

### 需求：现有技术检索策略

系统必须提供现有技术检索策略，支持专利申请前的检索工作。

#### 场景：生成检索关键词

- **当** 准备检索现有技术
- **那么** 为每项专利生成：
  - 核心技术关键词
  - 同义词和近义词
  - 上位概念和下位概念

#### 场景：推荐IPC分类号

- **当** 进行分类号检索
- **那么** 为每项专利推荐：
  - 主IPC分类号
  - 相关辅助分类号
  - 交叉检索分类号

#### 场景：识别竞品专利

- **当** 分析竞争态势
- **那么** 识别：
  - 主要竞争对手的专利布局
  - 潜在的在先技术风险
  - 规避设计建议
