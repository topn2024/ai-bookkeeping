## 需求
### 需求：专利申请材料管理

系统必须提供完整的专利申请材料管理功能，包括专利申请材料的创建、存储、检索和维护。

#### 场景：创建专利申请材料

- **当** 识别出具有专利价值的技术创新点
- **那么** 为每个技术创新点创建完整的专利申请材料，包括：
  - 专利申请书（发明名称、申请人信息、发明人信息、摘要）
  - 技术交底书（技术方案的详细描述）
  - 权利要求书（独立权利要求 + 从属权利要求）
  - 说明书（技术领域、背景技术、发明内容、具体实施方式）
  - 说明书附图（系统架构图、流程图、时序图）

#### 场景：专利申请材料存储

- **当** 完成专利申请材料的撰写
- **那么** 将所有材料存储在 `docs/patents/` 目录下
- **并且** 按照专利名称创建子目录，每个专利一个目录
- **并且** 目录结构为：
  ```
  docs/patents/
  ├── README.md（专利申请材料索引）
  ├── patent-01-multi-layer-intent-recognition/
  │   ├── application.md（专利申请书）
  │   ├── technical-disclosure.md（技术交底书）
  │   ├── claims.md（权利要求书）
  │   ├── specification.md（说明书）
  │   └── figures/（说明书附图）
  ├── patent-02-dual-channel-processing/
  │   └── ...
  └── ...
  ```

#### 场景：专利申请材料检索

- **当** 需要查找特定专利的申请材料
- **那么** 可以通过专利名称、技术领域、申请日期等信息进行检索
- **并且** 在 `README.md` 中提供专利清单和索引

#### 场景：专利申请材料更新

- **当** 收到专利审查意见或需要修改申请材料
- **那么** 可以更新对应专利目录下的文档
- **并且** 保留修改历史（通过Git版本控制）

### 需求：专利技术创新点识别

系统必须提供技术创新点识别功能，帮助识别具有专利价值的技术方案，并在申请前进行专利检索以确认无重复。

#### 场景：专利重复检查

- **当** 识别出潜在的技术创新点
- **那么** 必须先进行专利检索，确认是否已有相同或相似的专利
- **并且** 使用以下检索工具：
  - 国家知识产权局专利检索系统（http://pss-system.cnipa.gov.cn/）
  - Google Patents（https://patents.google.com/）
  - 专利之星（http://www.patentstar.com.cn/）
- **并且** 检索策略包括：
  - 关键词检索：使用技术方案的核心关键词
  - 分类号检索：使用IPC国际专利分类号（G06F、G10L、G06N、G06Q等）
  - 申请人检索：检索竞争对手的专利布局
  - 时间范围：检索近10年的专利（2015-2025）
- **并且** 根据检索结果决定：
  - 如果发现相同或高度相似的专利 → 放弃该技术方案的申请
  - 如果发现部分相似的专利 → 调整技术方案的描述，突出差异点
  - 如果未发现相似专利 → 继续撰写申请材料

#### 场景：代码库技术创新点分析

- **当** 需要识别代码库中的技术创新点
- **那么** 深入分析代码库的技术实现，重点关注：
  - 语音交互技术（意图识别、对话管理、打断检测等）
  - 多模态输入融合（图片、语音、文本的协同处理）
  - AI驱动的智能解析（账单解析、分类推荐、模型切换等）
  - 自学习系统（反馈采集、模式挖掘、置信度更新等）
  - 架构设计创新（解耦模式、并行处理、容错机制等）

#### 场景：技术创新点评估

- **当** 识别出潜在的技术创新点
- **那么** 评估其专利申请价值，考虑以下因素：
  - 新颖性：技术方案是否具有新颖性，与现有技术有何不同
  - 创造性：技术方案是否具有创造性，是否有技术难度
  - 实用性：技术方案是否具有实用性，是否能够实施
  - 商业价值：技术方案是否具有商业价值，是否值得保护

#### 场景：技术创新点分类

- **当** 完成技术创新点的识别和评估
- **那么** 将技术创新点分为不同优先级：
  - 核心专利（P0）：技术难度高、商业价值大、竞争优势明显
  - 重要专利（P1）：技术难度中等、商业价值中等、有一定竞争优势
  - 次要专利（P2）：技术难度较低、商业价值较低、可选择性申请

### 需求：专利申请文档规范

系统必须提供专利申请文档的规范和模板，确保申请材料符合《专利法》和《专利审查指南》的要求。

#### 场景：专利申请书撰写

- **当** 撰写专利申请书
- **那么** 必须包含以下内容：
  - 发明名称：简洁明确，不超过25个字
  - 申请人信息：公司名称、地址、邮编
  - 发明人信息：姓名、身份证号、联系方式
  - 摘要：技术方案的简要说明，200-300字

#### 场景：权利要求书撰写

- **当** 撰写权利要求书
- **那么** 必须包含以下内容：
  - 独立权利要求：1项，描述技术方案的核心特征
  - 从属权利要求：6-10项，分为3个层次细化保护范围
- **并且** 权利要求必须清楚、简洁，保护范围合理

#### 场景：说明书撰写

- **当** 撰写说明书
- **那么** 必须包含以下部分：
  - 技术领域：说明技术方案所属的技术领域
  - 背景技术：说明现有技术的状况和存在的问题
  - 发明内容：说明技术方案的目的、技术方案和有益效果
  - 附图说明：说明附图的内容
  - 具体实施方式：详细说明技术方案的实施方式，提供1-2个实施例
- **并且** 说明书必须充分公开技术方案，使本领域技术人员能够实施

#### 场景：说明书附图绘制

- **当** 绘制说明书附图
- **那么** 必须包含以下类型的图：
  - 系统架构图：展示系统的整体结构和模块关系
  - 流程图：展示技术方案的执行流程和逻辑关系
  - 时序图：展示多个模块之间的交互时序
- **并且** 附图必须清晰、简洁，不包含文字说明（除必要的标号）

### 需求：专利申请质量保证

系统必须提供专利申请质量保证机制，确保申请材料的质量和授权成功率。

#### 场景：专利检索

- **当** 完成专利申请材料的初稿
- **那么** 进行专利检索，确认技术方案的新颖性和创造性
- **并且** 使用以下检索工具：
  - 国家知识产权局专利检索系统
  - Google Patents
- **并且** 检索策略包括：
  - 关键词检索
  - 分类号检索（IPC分类号）
  - 申请人检索（竞争对手）
  - 引用检索

#### 场景：内部评审

- **当** 完成专利检索
- **那么** 组织内部评审，邀请技术团队和法务团队参与
- **并且** 评审内容包括：
  - 技术方案的准确性和完整性
  - 权利要求的保护范围是否合理
  - 说明书是否充分公开
  - 附图是否清晰
  - 是否符合专利法要求

#### 场景：外部审核（可选）

- **当** 内部评审完成
- **那么** 可以委托专利代理机构进行外部审核
- **并且** 根据审核意见优化申请材料

### 需求：专利申请提交

系统必须提供专利申请提交功能，支持向国家知识产权局提交专利申请。

#### 场景：准备提交材料

- **当** 完成所有审核和优化
- **那么** 准备提交材料清单：
  - 专利申请书
  - 权利要求书
  - 说明书
  - 说明书附图
  - 申请人身份证明（营业执照副本）
  - 发明人身份证明
  - 费用减免证明（如适用）

#### 场景：提交专利申请

- **当** 提交材料准备完成
- **那么** 可以选择以下提交方式：
  - 委托专利代理机构提交
  - 通过国家知识产权局电子申请系统自行提交
- **并且** 记录申请日期和申请号

#### 场景：跟踪审查进度

- **当** 专利申请提交后
- **那么** 定期跟踪审查进度
- **并且** 及时响应审查意见
- **并且** 记录审查过程和结果

### 需求：专利维护管理

系统必须提供专利维护管理功能，支持专利年费缴纳和专利价值评估。

#### 场景：专利年费提醒

- **当** 专利授权后
- **那么** 建立专利年费提醒机制
- **并且** 在年费到期前3个月发送提醒
- **并且** 记录年费缴纳情况

#### 场景：专利价值评估

- **当** 每年进行专利维护决策
- **那么** 评估每个专利的商业价值：
  - 技术相关性：技术是否仍在使用
  - 市场价值：技术是否有市场需求
  - 竞争优势：技术是否提供竞争优势
- **并且** 对于价值低的专利，可以选择放弃维护

#### 场景：专利档案管理

- **当** 专利申请、授权、维护过程中
- **那么** 维护完整的专利档案：
  - 申请材料
  - 审查意见和答复
  - 授权证书
  - 年费缴纳记录
  - 价值评估记录
- **并且** 所有档案存储在 `docs/patents/` 目录下
