## 5.5 傻瓜原则（Fool-Proof Design）

### 5.5.1 大白话原则 - 语言简化设计

**原则**：所有文案必须让小学三年级学生能够理解，禁止使用专业术语。

#### 5.5.1.1 词汇简化对照表

| 原专业术语 | 傻瓜化表达 | 说明 |
|-----------|----------|------|
| 预算 | 计划花多少钱 | 避免财务术语 |
| 结余 | 还剩多少钱 | 直白表达 |
| 支出 | 花了多少钱 | 口语化 |
| 收入 | 赚了多少钱 | 口语化 |

#### 5.5.1.3 语言简化服务

```dart
/// 傻瓜模式文案服务
class FoolProofTextService {
  /// 简化模式开关（可在设置中配置）
  final bool simplifiedMode;

  /// 词汇简化映射表
  static const Map<String, String> _simplifyMap = {
    '预算': '计划花多少钱',
    '结余': '还剩多少钱',
    '支出': '花的钱',
    '收入': '赚的钱',
    '同步': '存到云上',
    '备份': '存个副本',
    '钱龄': '钱放了多久',
    '资产': '家底',
    '负债': '欠的钱',
    // ... 更多映射
  };

  /// 获取简化文案
  String getSimplifiedText(String original) {
    if (!simplifiedMode) return original;

    String result = original;
    _simplifyMap.forEach((key, value) {
      result = result.replaceAll(key, value);
    });
    return result;
  }

  /// 生成傻瓜化金额描述
  String describeAmount(double amount, TransactionType type) {
    if (type == TransactionType.expense) {
      return '花了${amount.toInt()}块钱';
    } else {
      return '赚了${amount.toInt()}块钱';
    }
  }

  /// 生成傻瓜化日期描述
  String describeDate(DateTime date) {
    final now = DateTime.now();
    final diff = now.difference(date).inDays;

    if (diff == 0) return '今天';
    if (diff == 1) return '昨天';
    if (diff == 2) return '前天';
    if (diff < 7) return '$diff天前';
    if (diff < 30) return '${diff ~/ 7}个星期前';
    return '${date.month}月${date.day}号';
  }
}
```

class FoolProofDeleteConfirm {
  static Future<bool> confirm(
    BuildContext context, {
    required String itemDescription,
  }) async {
    // 第一次确认
    final firstConfirm = await showDialog<bool>(
      context: context,
      builder: (ctx) => AlertDialog(
        title: Text('要删掉吗？'),
        content: Text('这条账：$itemDescription'),
        actions: [

            onPressed: () => Navigator.pop(ctx, false),
            child: Text('不删了', style: TextStyle(fontSize: 18)),

            onPressed: () => Navigator.pop(ctx, true),
            child: Text('删掉', style: TextStyle(fontSize: 18)),

        ],
      ),
    );

    if (firstConfirm != true) return false;

    return await showDialog<bool>(
      context: context,

        ],
      ),

  }

### 5.5.4 即时反馈原则

**原则**：每一个操作都必须有明确的、即时的反馈，让用户知道发生了什么。

### 5.5.8 测试验收标准

| 测试项目 | 测试方法 | 合格标准 |
|---------|---------|---------|
| 文案理解度 | 找小学三年级学生测试 | 能复述文案含义 |
| 操作成功率 | 65岁以上老人测试 | 首次成功率≥80% |
| 任务完成时间 | 计时测试核心流程 | 记一笔账≤30秒 |
| 错误恢复率 | 故意制造错误场景 | 用户能自行恢复≥90% |
| 满意度评分 | 用后评分（1-5分） | 平均分≥4分 |
| 求助率 | 统计求助按钮点击 | 求助率≤5% |

### 5.5.9 与懒人设计原则的协同

傻瓜原则与懒人设计原则（第3章）是互补关系：

| 维度 | 懒人设计 | 傻瓜原则 |
|-----|---------|---------|
| 核心目标 | 减少操作步骤 | 降低理解门槛 |
| 服务对象 | 追求效率的用户 | 认知能力有限的用户 |
| 设计重点 | 一键完成、智能推荐 | 大白话、防呆设计 |
| 协同效果 | 既快又简单，人人能用 | 既快又简单，人人能用 |
