# 专利文档模板

本文档提供专利申请材料的标准模板，可直接复制使用。

## 1. 元数据文件模板 (metadata.json)

```json
{
  "patent_id": "P01",
  "patent_number": "",
  "title": {
    "zh": "中文专利名称",
    "en": "English Patent Title"
  },
  "inventors": ["发明人姓名"],
  "applicant": "申请人名称",
  "filing_date": "",
  "status": "drafting",
  "category": "core-technology",
  "tags": ["标签1", "标签2", "标签3"],
  "estimated_success_rate": "75-80%",
  "related_patents": [],
  "version": "2.0",
  "changelog": [
    {
      "version": "2.0",
      "date": "2026-01-18",
      "changes": "迁移到标准化目录结构"
    }
  ],
  "figures": [
    {
      "id": "fig-01",
      "title": "系统架构图",
      "type": "system-architecture",
      "file": "figures/fig-01-system-architecture.png"
    },
    {
      "id": "fig-02",
      "title": "算法流程图",
      "type": "algorithm-flow",
      "file": "figures/fig-02-algorithm-flow.png"
    },
    {
      "id": "fig-03",
      "title": "数据结构图",
      "type": "data-structure",
      "file": "figures/fig-03-data-structure.png"
    },
    {
      "id": "fig-04",
      "title": "时序图",
      "type": "sequence",
      "file": "figures/fig-04-sequence.png"
    }
  ],
  "files": {
    "specification": "specification.md",
    "claims": "claims.md",
    "abstract": "abstract.md"
  }
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patent_id | string | 是 | 专利编号，如 "P01" |
| patent_number | string | 否 | 专利申请号/授权号 |
| title.zh | string | 是 | 中文专利名称 |
| title.en | string | 是 | 英文专利名称 |
| inventors | array | 是 | 发明人列表 |
| applicant | string | 是 | 申请人名称 |
| filing_date | string | 否 | 申请日期（YYYY-MM-DD） |
| status | string | 是 | 状态：drafting/filed/granted |
| category | string | 是 | 分类：core-technology/user-experience/financial-management |
| tags | array | 是 | 技术标签 |
| estimated_success_rate | string | 是 | 预估成功率 |
| related_patents | array | 否 | 相关专利编号 |
| version | string | 是 | 文档版本号 |
| changelog | array | 是 | 版本变更记录 |
| figures | array | 是 | 附图列表 |
| files | object | 是 | 文件路径映射 |

## 2. 说明书模板 (specification.md)

```markdown
# [专利名称]

## 技术领域

[0001] 本发明涉及[技术领域]，具体涉及一种[具体技术方案简述]。

## 背景技术

[0002] [描述现有技术的状况]

[0003] [描述现有技术存在的问题]

[0004] [描述现有技术的不足之处]

## 发明内容

### 技术问题

[0005] 本发明要解决的技术问题是：[具体技术问题]

### 技术方案

[0006] 为解决上述技术问题，本发明提供一种[发明名称]，包括以下步骤/模块：

[0007] 步骤/模块一：[描述]

[0008] 步骤/模块二：[描述]

[0009] 步骤/模块三：[描述]

### 有益效果

[0010] 与现有技术相比，本发明具有以下有益效果：

[0011] 1. [效果一]

[0012] 2. [效果二]

[0013] 3. [效果三]

## 附图说明

[0014] 图1为[图1标题]；

[0015] 图2为[图2标题]；

[0016] 图3为[图3标题]；

[0017] 图4为[图4标题]。

## 具体实施方式

[0018] 下面结合附图和具体实施例对本发明作进一步详细说明。

### 实施例一

[0019] 如图1所示，[详细描述实施例一的技术方案]

[0020] [继续描述具体实现细节]

### 实施例二

[0021] [描述另一种实施方式]

## 技术效果验证

[0022] [描述实验数据或性能对比]

---

**注意事项**：
1. 段落编号使用 [XXXX] 格式，从 0001 开始连续编号
2. 技术方案描述要具体、可实施
3. 有益效果要与技术方案对应
4. 附图说明要与实际附图一致
5. 具体实施方式要详细，能够指导本领域技术人员实施
```

## 3. 权利要求书模板 (claims.md)

```markdown
# 权利要求书

## 独立权利要求

1. 一种[发明名称]，其特征在于，包括以下步骤/模块：

   步骤/模块S1：[核心技术特征一]；

   步骤/模块S2：[核心技术特征二]；

   步骤/模块S3：[核心技术特征三]。

## 从属权利要求

2. 根据权利要求1所述的[发明名称]，其特征在于，所述步骤S1中，[进一步限定技术特征]。

3. 根据权利要求1所述的[发明名称]，其特征在于，所述步骤S2中，[进一步限定技术特征]。

4. 根据权利要求1所述的[发明名称]，其特征在于，所述步骤S3包括：

   S3.1：[子步骤一]；

   S3.2：[子步骤二]。

5. 根据权利要求1至4任一项所述的[发明名称]，其特征在于，还包括[附加技术特征]。

6. 根据权利要求1所述的[发明名称]，其特征在于，所述[某技术特征]采用[具体实现方式]。

7. 根据权利要求1所述的[发明名称]，其特征在于，所述[某参数]的取值范围为[具体范围]。

8. 一种实现权利要求1-7任一项所述方法的系统，其特征在于，包括：

   [模块一]：用于[功能描述]；

   [模块二]：用于[功能描述]；

   [模块三]：用于[功能描述]。

9. 一种计算机可读存储介质，其特征在于，所述存储介质存储有计算机程序，所述计算机程序被处理器执行时实现权利要求1-7任一项所述的方法。

10. 一种电子设备，其特征在于，包括：

    处理器；

    存储器，用于存储计算机程序；

    其中，所述处理器执行所述计算机程序时实现权利要求1-7任一项所述的方法。

---

**权利要求书撰写要点**：

1. **独立权利要求**（第1条）：
   - 保护范围最大
   - 只包含必要技术特征
   - 使用"其特征在于"分隔前序部分和特征部分

2. **从属权利要求**（第2-7条）：
   - 引用独立权利要求或其他从属权利要求
   - 逐步细化技术特征
   - 分层次扩展保护范围

3. **系统权利要求**（第8条）：
   - 与方法权利要求对应
   - 描述实现方法的系统模块

4. **存储介质权利要求**（第9条）：
   - 保护软件实现形式

5. **设备权利要求**（第10条）：
   - 保护硬件实现形式
```

## 4. 摘要模板 (abstract.md)

```markdown
# 摘要

## 发明名称

[中文专利名称]

## 摘要内容

本发明公开了一种[发明名称]，属于[技术领域]。该方法/系统包括：[核心步骤/模块简述]。与现有技术相比，本发明[主要优点1]，[主要优点2]，[主要优点3]。本发明可应用于[应用场景]，具有[实用价值]。

## 摘要附图

图[X]

---

**摘要撰写要点**：

1. 字数控制在 200-300 字
2. 包含技术领域、技术方案、有益效果
3. 不使用商业性宣传用语
4. 摘要附图选择最能体现技术方案的一张图
```

## 5. 目录结构模板

创建新专利时，复制以下目录结构：

```
P{XX}-{name}/
├── metadata.json           # 复制元数据模板并修改
├── specification.md        # 复制说明书模板并修改
├── claims.md              # 复制权利要求书模板并修改
├── abstract.md            # 复制摘要模板并修改
├── figures/               # 创建流程图目录
│   ├── fig-01-system-architecture.svg
│   ├── fig-01-system-architecture.png
│   ├── fig-02-algorithm-flow.svg
│   ├── fig-02-algorithm-flow.png
│   ├── fig-03-data-structure.svg
│   ├── fig-03-data-structure.png
│   ├── fig-04-sequence.svg
│   └── fig-04-sequence.png
└── supplementary/         # 可选：补充材料
    ├── experiments.md
    ├── comparisons.md
    └── references.md
```

## 6. 使用说明

1. 复制相应模板到目标文件
2. 替换 `[占位符]` 为实际内容
3. 删除模板中的说明性注释
4. 确保段落编号连续
5. 检查所有交叉引用正确
