# 专利文档命名规范

本文档定义了专利文档的标准命名规则，确保所有专利文档易于管理和查找。

## 1. 专利目录命名

### 1.1 格式

```
P{两位数字编号}-{中文简称}
```

### 1.2 规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 编号格式 | 两位数字，从01开始 | P01, P02, ..., P18 |
| 中文简称 | 4-8个汉字，简洁明了 | FIFO钱龄计算 |
| 编号连续 | 不允许跳号 | P01, P02, P03... |

## 2. 文件命名

### 2.1 必需文件

| 文件名 | 用途 | 格式 |
|--------|------|------|
| `metadata.json` | 专利元数据 | JSON |
| `specification.md` | 说明书（完整版） | Markdown |
| `claims.md` | 权利要求书 | Markdown |
| `abstract.md` | 摘要 | Markdown |

### 2.2 流程图文件

```
fig-{两位数字}-{英文描述}.{扩展名}
```

| 规则 | 说明 | 示例 |
|------|------|------|
| 编号 | 两位数字，从01开始 | fig-01, fig-02 |
| 描述 | 小写字母，连字符分隔 | system-architecture |
| 扩展名 | mmd（源文件）、svg或png | .mmd, .svg, .png |

**标准流程图命名：**

| 图号 | 文件名 | 用途 |
|------|--------|------|
| 图1 | fig-01-system-architecture | 系统架构图 |
| 图2 | fig-02-algorithm-flow | 算法流程图 |
| 图3 | fig-03-data-structure | 数据结构图 |
| 图4 | fig-04-sequence | 时序图 |

## 3. 目录结构示例

```
docs/patents/
├── README.md                           # 专利组合总览
├── index.json                          # 专利索引
├── validation-report.md                # 验证报告
├── standards/                          # 标准规范
│   ├── naming-convention.md           # 本文档
│   ├── figure-standards.md            # 流程图标准
│   └── document-template.md           # 文档模板
├── archive/                            # 归档旧版本
└── patents/                            # 当前版本专利

    └── ...
```

## 4. 命名检查清单

在创建或迁移专利文档时，请检查：

- [ ] 目录名称格式为 `P{XX}-{中文简称}`
- [ ] 编号为两位数字且连续
- [ ] 中文简称4-8个字，简洁明了
- [ ] 必需文件名称正确（metadata.json, specification.md, claims.md, abstract.md）
- [ ] 流程图命名格式为 `fig-{XX}-{描述}.{扩展名}`
- [ ] 流程图同时存在 mmd 和 svg 格式

## 5. 版本管理

- 版本号格式：`X.Y`（如 1.0, 1.1, 2.0）
- 版本号记录在 `metadata.json` 中
- 不在文件名中包含版本号
- 旧版本归档到 `archive/` 目录
