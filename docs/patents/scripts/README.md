# 专利现有技术检索工具使用说明

**创建日期**: 2026-01-19
**工具版本**: v1.0

---

## 1. 工具概述

本工具包提供两个自动化检索工具，帮助完成18个专利的现有技术检索工作：

1. **Python命令行工具** (`patent_search_automation.py`)
   - 自动加载检索策略
   - 生成检索查询
   - 分析文献相关性
   - 生成检索报告

2. **Web检索助手** (`search-assistant.html`)
   - 可视化检索界面
   - 一键复制检索式
   - 记录检索结果
   - 本地存储检索记录

## 2. Python命令行工具使用

### 2.1 基本用法

```bash
# 进入scripts目录
cd /Users/beihua/code/baiji/ai-bookkeeping/docs/patents/scripts

# 查看专利检索策略
python3 patent_search_automation.py P05

# 查看其他专利
python3 patent_search_automation.py P11
python3 patent_search_automation.py P12
python3 patent_search_automation.py P14
python3 patent_search_automation.py P18
```

### 2.2 输出示例

```
=== P05 检索策略 ===

数据库: CNIPA, CNKI, Google Scholar, Google Patents

检索式数量: 8

中文关键词: 13个
英文关键词: 10个

=== 生成检索查询 ===

CNIPA:
  1. (大语言模型 OR LLM OR GPT OR BERT OR Transformer) AND ...
  2. (语音记账 OR 智能记账 OR 自动记账) AND ...
  3. (语音识别 OR ASR) AND (大语言模型 OR LLM) AND ...

CNKI:
  1. (大语言模型 + LLM + GPT + BERT + Transformer) * ...
  2. (语音记账 + 智能记账 + 自动记账) * ...
  3. (语音识别 + ASR) * (大语言模型 + LLM) * ...

...
```

### 2.3 功能说明

**自动加载检索策略**:
- 从`reviews/search-strategies/`目录读取检索策略文件
- 提取检索式、关键词、数据库信息

**生成检索查询**:
- 为不同数据库生成适配的检索式
- CNIPA: 使用AND/OR逻辑
- CNKI: 转换为+/*格式
- Google Scholar: 转换为|格式

**分析文献相关性**:
- 基于关键词匹配计算相关性评分
- 评分范围: 0-1

**生成检索报告**:
- 自动生成Markdown格式报告
- 包含检索概况、记录、文献列表等

## 3. Web检索助手使用

### 3.1 打开工具

```bash

open scripts/search-assistant.html

# 方法2: 使用文件路径

```

### 3.2 使用流程

**步骤1: 选择专利**
- 点击专利按钮（如"P05 - LLM语音交互"）
- 查看专利完整名称和信息

**步骤2: 查看检索式**
- 自动显示该专利的所有检索式
- 点击"复制"按钮复制检索式到剪贴板

**步骤3: 打开数据库**
- 点击数据库卡片中的"打开检索系统"按钮
- 在新标签页中打开对应数据库

**步骤4: 执行检索**
- 在数据库中粘贴检索式
- 执行检索并查看结果

**步骤5: 记录结果**
- 返回Web检索助手
- 填写检索记录表单:
  - 数据库: 选择使用的数据库
  - 检索式: 输入使用的检索式
  - 结果总数: 输入检索结果总数
  - 相关文献数: 输入相关文献数量
  - 备注: 记录发现和问题
- 点击"保存记录"

**步骤6: 查看记录**
- 自动显示已保存的检索记录
- 记录保存在浏览器本地存储中

### 3.3 功能特点

**可视化界面**:
- 清晰的分步指导
- 一键复制检索式
- 直接链接到数据库

**本地存储**:

**多专利支持**:

## 4. 检索数据库说明

### 4.1 国家知识产权局

**网址**: http://pss-system.cnipa.gov.cn

**检索方法**:
1. 选择"高级检索"
2. 在"摘要"或"全文"字段输入检索式
3. 使用AND、OR、NOT逻辑运算符
4. 点击"检索"

**注意事项**:
- 需要注册账号
- 支持中文检索
- 可导出检索结果

### 4.2 CNKI（中国知网）

**网址**: https://www.cnki.net

**检索方法**:
1. 选择"高级检索"
2. 在"主题"或"摘要"字段输入检索式
3. 使用*（AND）、+（OR）逻辑运算符
4. 点击"检索"

**注意事项**:
- 需要机构账号或付费
- 支持中英文检索
- 可导出文献列表

### 4.3 万方数据库

**网址**: https://www.wanfangdata.com.cn

**检索方法**:
1. 选择"高级检索"
2. 在"主题"或"摘要"字段输入检索式
3. 使用AND、OR逻辑运算符
4. 点击"检索"

**注意事项**:
- 需要机构账号或付费
- 支持中文检索
- 可导出文献列表

### 4.4 Google Scholar

**网址**: https://scholar.google.com

**检索方法**:
1. 在搜索框输入检索式
2. 使用|（OR）、-（NOT）逻辑运算符
3. 使用引号""进行精确匹配
4. 点击"搜索"

**注意事项**:
- 免费使用
- 支持中英文检索
- 可查看引用次数

### 4.5 Google Patents

**网址**: https://patents.google.com

**检索方法**:
1. 在搜索框输入检索式
2. 使用AND、OR、NOT逻辑运算符
3. 可按国家、日期筛选
4. 点击"搜索"

**注意事项**:
- 免费使用
- 主要为英文专利
- 可查看专利家族

### 6.2 报告结构

生成的报告包含以下部分：

1. **检索概况**
   - 检索数据库列表
   - 检索式数量
   - 检索结果统计

2. **检索记录**
   - 每次检索的详细记录
   - 数据库、检索式、结果数

3. **相关文献列表**
   - 高度相关文献
   - 中度相关文献
   - 低度相关文献

4. **新颖性评估**
   - 对比分析
   - 差异点总结
   - 新颖性结论

5. **创造性评估**
   - 最接近现有技术
   - 区别特征
   - 技术效果
   - 创造性结论

6. **综合评估**
   - 授权可行性
   - 风险提示
   - 优化建议

## 7. 工作流程建议

### 7.1 第一优先级专利（P05, P11, P12, P14, P18）

**第1周**:
- 每天检索1个专利
- 每个专利检索时间: 12-14小时
- 使用Web助手记录结果

**检索顺序**:
1. 周一: P14（冷静期控制，风险最低）
2. 周二: P18（消费趋势预测，风险最低）
3. 周三: P05（LLM语音交互）
4. 周四: P11（离线增量同步）
5. 周五: P12（游戏化激励）

### 7.2 每个专利的检索流程

**上午（4小时）**:
1. 国家知识产权局检索（3小时）
   - 使用3个检索式
   - 筛选相关专利
   - 记录检索结果
2. 休息（1小时）

**下午（4小时）**:
1. CNKI学术文献检索（2小时）
   - 使用2个检索式
   - 筛选相关论文
   - 记录检索结果
2. Google Scholar检索（2小时）
   - 使用英文检索式
   - 筛选国际论文
   - 记录检索结果

**晚上（4小时）**:
1. Google Patents检索（2小时）
   - 使用英文检索式
   - 筛选国际专利
   - 记录检索结果
2. 整理检索结果（2小时）
   - 汇总所有记录
   - 初步分析相关性
   - 标记高度相关文献

### 7.3 检索技巧

**扩展检索**:
- 如果结果太少（<10篇），放宽检索条件
- 如果结果太多（>1000篇），增加限制条件

**引文检索**:
- 查看高度相关文献的引用文献
- 查看高度相关文献的被引文献

**申请人检索**:
- 检索竞争对手的相关专利
- 重点关注阿里巴巴、腾讯、字节跳动等

**时间筛选**:
- 优先检索近5年的文献（2020-2025）
- 对于基础技术，可扩展到10年

## 8. 常见问题

### Q1: 检索式太复杂，数据库不支持怎么办？

**A**: 简化检索式，分步检索：
1. 先使用核心关键词检索
2. 在结果中再次筛选
3. 使用数据库的高级筛选功能

### Q2: 检索结果太多，如何筛选？

**A**: 使用以下方法筛选：
1. 按时间筛选（近5年）
2. 按引用次数筛选（高引用）
3. 按相关性排序
4. 阅读标题和摘要快速判断

### Q3: 如何判断文献的相关性？

**A**: 使用以下标准：
- **高度相关**: 技术方案高度相似，可能影响新颖性
- **中度相关**: 部分技术特征相似，可能影响创造性
- **低度相关**: 仅技术领域相关，影响较小

### Q4: Web助手的记录丢失了怎么办？

**A**:
1. 检查浏览器localStorage是否被清除
2. 定期导出记录到JSON文件
3. 使用Python工具作为备份

### Q5: 如何处理检索到的外文文献？

**A**:
1. 使用Google翻译或DeepL翻译标题和摘要
2. 重点关注技术方案和附图
3. 必要时请专业翻译协助
