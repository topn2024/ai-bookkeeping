# 权利要求书

## 智能财务健康评分与管理方法及系统

### 独立权利要求

**权利要求1**：一种智能财务健康评分与管理方法，其特征在于，包括以下步骤：

S1. 多维度评分步骤：
- 基于六个维度进行综合评分：收支平衡（20分）、预算执行（20分）、储蓄能力（15分）、债务健康（15分）、消费结构（15分）、目标达成（15分）；
- 综合评分 S = Σ(维度分数)，满分100分；
- 划分健康等级：优秀（90-100）、良好（80-89）、中等（70-79）、较差（60-69）、差（0-59）；

S2. 智能预警步骤：
- 提供四级预警机制：正常（绿色）、提示（黄色）、警告（橙色）、严重（红色）；
- 根据消费进度与时间进度差异判定预警级别；
- 消费进度 > 时间进度 + 20%时为严重预警；

S3. 问题诊断与建议生成步骤：
- 自动识别五类问题：收支失衡、预算超支、储蓄不足、债务过高、消费结构不合理；
- 生成分层建议：短期（1个月）、中期（3个月）、长期（1年）；
- 按优先级排序建议：紧急且重要、重要不紧急、紧急不重要、不紧急不重要；

S4. 目标追踪与激励步骤：
- 智能分解长期目标为月度、周度、日度目标；
- 实时追踪目标完成度；
- 提供多维激励：进度激励、超额激励、连续激励、挽救激励。

**权利要求2**：一种智能财务健康评分与管理系统，其特征在于，包括：

多维度评分模块：用于基于六维度计算综合健康评分并划分等级；

智能预警模块：用于提供四级预警和预测性预警；

建议生成模块：用于诊断问题并生成分层个性化建议；

目标追踪模块：用于分解目标、追踪进度和提供激励反馈。

### 从属权利要求

**权利要求3**：根据权利要求1所述的方法，其特征在于，所述收支平衡评分计算为：

- 结余率 = (收入 - 支出) / 收入；
- 结余率>20%：20分；
- 结余率10-20%：15分；
- 结余率0-10%：10分；
- 结余率<0%：0分。

**权利要求4**：根据权利要求1所述的方法，其特征在于，所述债务健康评分计算为：

- 债务收入比 = 总债务 / 月收入；
- 无债务：15分；
- 债务收入比<3：12分；
- 债务收入比3-6：8分；
- 债务收入比>6：3分。

**权利要求5**：根据权利要求1所述的方法，其特征在于，所述预测性预警包括：

- 计算日均消费 = 已消费金额 / 已过天数；
- 预计耗尽天数 = 剩余预算 / 日均消费；
- 预计耗尽日期 = 今天 + 预计耗尽天数；
- 预计耗尽天数<7时触发预警。

**权利要求6**：根据权利要求1所述的方法，其特征在于，所述智能提醒策略包括：

- 正常级别：不提醒；
- 提示级别：每周1次提醒；
- 警告级别：每3天1次提醒；
- 严重级别：每天1次提醒；
- 在用户活跃时段发送提醒。

**权利要求7**：根据权利要求1所述的方法，其特征在于，所述多维激励包括：

- 进度激励：完成25%、50%、75%、100%时分别触发；
- 超额激励：超额完成10%、20%时触发；
- 连续激励：连续3天、7天、30天达标时触发；
- 挽救激励：从落后状态追赶到达标时触发。

**权利要求8**：根据权利要求1所述的方法，其特征在于，还包括趋势分析步骤：

- 计算近3个月评分趋势；
- 识别改善或恶化趋势；
- 预测未来1个月评分；
- 趋势恶化时提前预警。

**权利要求9**：根据权利要求2所述的系统，其特征在于，还包括：

- 可视化模块：展示六维雷达图、趋势曲线和目标进度条；
- 历史分析模块：回顾历史健康评分和改善轨迹；
- 对比分析模块：与同类用户匿名对比（在用户授权下）。
