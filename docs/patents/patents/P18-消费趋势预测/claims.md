# 权利要求书

## 基于中国消费市场特征的多算法融合趋势预测方法及系统

### 独立权利要求

**权利要求1**：一种基于中国消费市场特征的多算法融合趋势预测方法，其特征在于，包括以下步骤：

S1. 多算法融合预测步骤：
- 采用简单移动平均、加权移动平均、指数移动平均和线性回归四种算法进行并行预测；
- 使用固定权重0.2、0.3、0.2、0.3进行融合，得到基础预测值；
- 公式：base_prediction = 0.2×SMA + 0.3×WMA + 0.2×EMA + 0.3×trend；

S2. 中国市场季节性调整步骤：
- 计算12个月季节性因子，公式：seasonal_factor[month] = monthly_avg[month] / yearly_avg；
- 应用中国市场特殊因子：春节（1月底-2月）1.3、618促销（6月）1.2、双11（11月）1.4、双12及年终（12月）1.2；
- 平滑处理：将因子裁剪到[0.7, 1.5]范围；
- 最终预测：final_prediction = base_prediction × seasonal_factor；

S3. 置信区间计算步骤：
- 计算历史预测误差的标准差 std_error；
- 使用正态分布95%置信度，z_score = 1.96；
- 置信区间：[prediction - 1.96×std_error, prediction + 1.96×std_error]；

S4. 消费洞察生成步骤：
- 检测拿铁因子：小额高频消费占比超过30%时生成节约建议；
- 分析周末消费模式：周末消费占比超过50%时建议周末预算规划；
- 识别异常大额消费：单笔超过1000元时提醒提前规划；
- 生成趋势分析：月均增长超过10%时警示消费增长。

**权利要求2**：一种基于中国消费市场特征的多算法融合趋势预测系统，其特征在于，包括：

多算法融合模块：用于并行执行四种预测算法并融合结果；

中国市场季节性模块：用于计算和应用包含中国特有促销节日的季节性因子；

置信区间模块：用于计算95%置信区间并评估预测可靠性；

消费洞察模块：用于分析消费模式并生成可操作的建议。

### 从属权利要求

**权利要求3**：根据权利要求1所述的方法，其特征在于，所述中国市场特殊因子还包括：

- 五一劳动节（5月）：1.15；
- 国庆节（10月）：1.25；
- 中秋节（农历八月十五对应公历月份）：1.1；
- 因子动态更新：new_factor = 0.7 × old_factor + 0.3 × actual_factor。

**权利要求4**：根据权利要求1所述的方法，其特征在于，所述置信度评估包括：

- 数据质量评分：data_quality = min(历史月数 / 12, 1.0)；
- 稳定性评分：stability = 1 - min(变异系数, 1.0)；
- 综合置信度：confidence = 0.5 × data_quality + 0.5 × stability；
- 置信等级：高（>0.8）、中（0.6-0.8）、低（<0.6）。

**权利要求5**：根据权利要求1所述的方法，其特征在于，还包括预算超支预警步骤：

- 计算消费速度：spending_rate = 当前消费 / 已过天数；
- 预测月末消费：projected = spending_rate × 当月总天数；
- 进度比较：消费进度 > 时间进度 + 0.2时为严重预警，+ 0.1时为警告；
- 建议限额：daily_limit = (预算 - 当前消费) / 剩余天数；
- 提前预警：剩余7天且预计超支时发送通知。

**权利要求6**：根据权利要求1所述的方法，其特征在于，所述拿铁因子检测包括：

- 小额消费定义：单笔金额小于50元；
- 高频判定：月均超过20笔小额消费；
- 节约计算：saved_amount = 小额消费总额 × 0.5；
- 洞察文本：生成"每月可节省{saved_amount}元"的建议。

**权利要求7**：根据权利要求1所述的方法，其特征在于，还包括动态权重调整步骤：

- 回测评估：计算各算法在近3个月的预测误差；
- 权重更新：误差小的算法增加权重，误差大的算法减少权重；
- 权重约束：任一算法权重在[0.1, 0.5]范围内；
- 更新频率：每月更新一次。

**权利要求8**：根据权利要求1所述的方法，其特征在于，还包括异常值处理步骤：

- 检测方法：z_score = (value - mean) / std，|z_score| > 3为异常；
- 处理方式：使用中位数替代异常值；
- 标记保留：保留异常标记用于洞察分析。

**权利要求9**：根据权利要求1所述的方法，其特征在于，还包括个性化消费模式学习步骤：

- 周期消费识别：检测固定周期（周、月、季度）的重复消费模式；
- 固定支出提取：识别房租、会员订阅等固定金额支出；
- 弹性支出分离：将总消费分为固定支出和弹性支出分别预测；
- 预测公式：total_prediction = fixed_prediction + elastic_prediction。

**权利要求10**：根据权利要求2所述的系统，其特征在于，还包括：

- 分类预测模块：对餐饮、购物、娱乐等分类分别进行趋势预测；
- 可视化模块：展示预测趋势线、置信区间阴影和历史数据对比；
- 反馈学习模块：根据实际消费数据持续优化模型参数。
