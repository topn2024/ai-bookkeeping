# 权利要求书

## 基于周期性消费识别的订阅管理和浪费检测方法及系统

### 独立权利要求

**权利要求1**：一种基于周期性消费识别的订阅管理方法，其特征在于，包括以下步骤：

S1. 周期性消费识别步骤：
- 按商户名称和金额（允许±5%误差）分组，形成候选订阅组；
- 计算相邻交易的时间间隔；
- 判定周期类型：周（7±2天）、双周（14±3天）、月（30±5天）、季（90±10天）、年（365±15天）；
- 周期性确认：至少3次交易，间隔变异系数<0.2；

S2. 稳定性评分步骤：
- 金额稳定性（40分）：基于金额变异系数；
- 时间稳定性（40分）：基于间隔变异系数；
- 持续性（20分）：基于订阅持续月数；
- 综合评分 = 金额稳定性 + 时间稳定性 + 持续性；

S3. 使用状态评估步骤：
- 直接使用数据：观看次数、播放次数、打卡次数；
- 间接推断：关联消费分析；
- 时间衰减模型：usage_probability = exp(-days_since_last_use / 30)；
- 使用状态分级：经常使用（每周≥2次）、偶尔使用（每月1-4次）、很少使用（每季度1-2次）、完全未使用（3个月无记录）；

S4. 浪费订阅检测步骤：
- 完全未使用：3个月无使用记录，判定为浪费；
- 很少使用：单次成本>单次购买成本×2，判定为浪费；
- 重复订阅：同类服务订阅多个，判定为浪费；
- 计算年度节省潜力 = Σ(浪费订阅的年度成本)；

S5. 智能续费提醒步骤：
- 在续费前7天、3天、1天发送提醒；
- 提供使用状态、浪费评估、操作建议。

**权利要求2**：一种基于周期性消费识别的订阅管理系统，其特征在于，包括：

周期性识别模块：用于自动识别周期性消费；

稳定性分析模块：用于评估订阅稳定性；

使用评估模块：用于评估订阅使用状态；

浪费检测模块：用于识别浪费订阅并计算潜在节省；

智能提醒模块：用于发送续费提醒和操作建议。

### 从属权利要求

**权利要求3**：根据权利要求1所述的方法，其特征在于，所述周期性确认条件为：

- 至少3次交易；
- 间隔标准差/平均间隔 < 0.2；
- 最近一次交易在3个周期内。

**权利要求4**：根据权利要求1所述的方法，其特征在于，所述浪费检测还包括：

- 功能重叠：不同服务功能重叠，判定为部分浪费；
- 价格不合理：价格>市场价×1.5，判定为浪费。

**权利要求5**：根据权利要求1所述的方法，其特征在于，所述智能提醒的建议规则为：

- 完全未使用→强烈建议取消；
- 很少使用→建议按需购买；
- 偶尔使用→考虑降级套餐；
- 经常使用→建议继续订阅。

**权利要求6**：根据权利要求1所述的方法，其特征在于，还包括用户反馈学习步骤：

- 记录用户决策（继续/取消）；
- 调整浪费检测阈值；
- 优化提醒策略。

**权利要求7**：根据权利要求1所述的方法，其特征在于，还包括中国市场特殊处理：

- 视频会员：考虑热播剧集周期；
- 外卖会员：考虑工作日vs周末；
- 共享单车：考虑季节因素（冬天使用少）。

**权利要求8**：根据权利要求1所述的方法，其特征在于，所述稳定性等级分为：

- 高度稳定：≥80分；
- 较稳定：60-79分；
- 中等稳定：40-59分；
- 不稳定：<40分。

**权利要求9**：根据权利要求2所述的系统，其特征在于，还包括：

- 数据采集模块：收集交易数据；
- 可视化模块：展示订阅列表、使用状态、节省金额；
- 学习模块：根据用户反馈优化检测策略。
