# 软件界面截图指南

## 一、截图要求

### 1. 技术要求
- **格式**：PNG（推荐）或JPG
- **分辨率**：
  - 移动端：1080x1920（竖屏）或更高
  - 管理后台：1920x1080（横屏）或更高
- **文件大小**：每张截图不超过5MB
- **清晰度**：确保文字和图标清晰可辨

### 2. 内容要求
- 展示真实的软件界面
- 包含测试数据（非真实用户数据）
- 无敏感信息（API密钥、真实用户信息等）
- 体现软件的核心功能和特色

## 三、截图步骤

### 方法1：使用模拟器截图（推荐）

1. **启动应用**
   ```bash
   cd app
   flutter run
   ```

2. **准备测试数据**
   - 创建测试账本
   - 添加测试交易记录
   - 设置测试预算
   - 确保数据丰富且有代表性

3. **截图**
   - Android模拟器：使用模拟器工具栏的截图按钮
   - iOS模拟器：使用 `Cmd + S` 快捷键
   - 或使用 `flutter screenshot` 命令

4. **保存截图**
   - 截图保存到 `软著材料/界面截图/` 目录
   - 命名格式：`01-首页界面.png`、`02-图片识别记账.png` 等

### 方法2：使用真机截图

1. **连接真机**
   ```bash
   flutter devices
   flutter run -d <device-id>
   ```

2. **截图**
   - Android：电源键 + 音量下键
   - iOS：侧边按钮 + 音量上键

3. **传输截图**
   - 通过USB传输到电脑
   - 或使用AirDrop（iOS）

### 方法3：使用截图工具

推荐工具：
- **Snipaste**（Windows/Mac）
- **截图**（Mac自带）
- **Flameshot**（Linux）

## 四、截图后处理

### 1. 检查清单
- [ ] 分辨率是否符合要求
- [ ] 图片是否清晰
- [ ] 是否包含敏感信息
- [ ] 文件大小是否合理
- [ ] 文件命名是否规范

### 2. 敏感信息处理
如果截图中包含敏感信息，使用图片编辑工具进行处理：
- 模糊处理（推荐）
- 马赛克
- 纯色遮盖

**需要处理的敏感信息**：
- 真实用户姓名
- 真实手机号
- 真实邮箱地址
- API密钥
- 真实交易金额（如果涉及隐私）

## 六、常见问题

### Q1：截图模糊怎么办？
A：确保使用高分辨率设备或模拟器，避免缩放截图。

### Q2：如何添加测试数据？
A：在应用中手动创建测试账本和交易记录，或使用数据导入功能。

### Q3：截图中的数据需要真实吗？
A：不需要，使用测试数据即可，但要确保数据合理且有代表性。

### Q4：需要截取所有功能吗？
A：不需要，重点截取核心功能和创新特色功能，6-8张即可。

### Q5：截图可以编辑吗？
A：可以进行必要的编辑（如模糊敏感信息），但不要过度美化或添加不存在的功能。
